# Reflection 18 – The Portfolio Is Also the Project

## Date
Retrospective reflection based on final submission preparation

## Model
Rolfe et al. – What? So What? Now What?

## What?

Near the end, I realised that I had been thinking too narrowly. I thought the project was mainly the backend code. Then I looked again at the checklist and understood that the portfolio itself is part of the project.

The code is important, but the assessor also needs to see tests, screenshots, diagrams, reflections, analysis, research, references, decisions, limitations and the final report. At first, this felt like extra work. Then I realised it is how the project becomes understandable.

## So What?

This mattered because a project that only exists in my head is not assessable. The tutor cannot guess my struggle, my decisions or my learning. I have to show it.

The portfolio became the story of the project. The screenshots show what happened. The tests show whether it worked. The reflections show what I learned. The analysis shows what the results mean.

## Now What?

In future, I will create evidence from the first week, not the last week. Every time I solve a problem, I will save a screenshot, write a small note and commit it. That way, the portfolio will grow naturally with the project.
