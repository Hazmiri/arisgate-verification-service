# Reflection 10 – Final Portfolio Packaging

## Date
27 April 2026

## Model
Rolfe et al. – What? So What? Now What?

## What?
Near the end of the project, I realised that finishing the code was not the same as finishing the submission. I needed to organise screenshots, write test notes, create a master test matrix, update the README, add analysis, and prepare reflections. This made the project feel more serious because the evidence had to make sense to someone else.

## So What?
This changed how I understood software development. A project is not only what runs on the machine. It is also how clearly the work can be explained, tested, justified, and improved. The final portfolio became a way to show my process, not just my final output.

## Now What?
Before submission, I will focus on clarity rather than adding new features. I will make sure the assessor can follow the project quickly, understand the evidence, and see that the prototype is working, tested, and honestly evaluated.
