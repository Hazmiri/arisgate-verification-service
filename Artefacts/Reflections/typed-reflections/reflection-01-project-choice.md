# Reflection 01 – Choosing the COD Verification Problem

## Date
Retrospective reflection written during final submission preparation

## Model
Rolfe et al. – What? So What? Now What?

## What?
When I first chose the project, I did not want to build something that felt disconnected from real life. I kept thinking about Cash on Delivery because I had seen how E-commerce can be affected by fake orders, weak customer details, failed deliveries, and uncertainty before dispatch. At the beginning, I was not fully sure how to turn that problem into software, but I knew the idea was practical.

## So What?
This mattered because the project became more than an academic task. It connected with my own experience, my interest in online business, and my aim to become a stronger developer. I realised that a good software project starts with a real problem, not only with code. ArisGate gave me a reason to learn backend development, APIs, databases, testing, and security in one connected project.

## Now What?
In future projects, I will start by defining the problem clearly before choosing technologies. I now understand that code is stronger when it is connected to a user, a business need, or a practical situation.
