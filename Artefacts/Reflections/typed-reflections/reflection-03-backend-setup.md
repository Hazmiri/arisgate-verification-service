# Reflection 03 – Backend Setup and Early Debugging

## Date
Retrospective reflection written during final submission preparation

## Model
Rolfe et al. – What? So What? Now What?

## What?
During the early backend setup, I made several mistakes that felt small but stopped the whole server from running. I typed `noemon` instead of `nodemon`, wrote `dontenv` instead of `dotenv`, and had dependency errors such as missing `cors`. At the time, I felt confused because each error looked serious.

## So What?
This helped me understand that development is often about reading errors calmly. The terminal was not attacking me; it was giving evidence. I also learned that one wrong letter can break a Node.js application, and that backend development requires accuracy in commands, imports, package names, and file paths.

## Now What?
Now I check spelling, folder location, and package installation before assuming the project is broken. In future work, I will use errors as clues and debug one step at a time instead of panicking or changing many things at once.
